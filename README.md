# Engineering Review Platform

An autonomous multi-agent AI system that reviews code repositories and pull requests, posts inline GitHub comments, tracks developer progress over time, and runs scheduled health scans — all running locally via Docker.

![Dashboard preview](https://placehold.co/800x400/0f0f1a/6366f1?text=Engineering+Review+Platform)

---

## What it does

- **Full repo reviews** — 11-stage LangGraph pipeline (security, architecture, testing, scalability, tech debt) with a coaching report tailored to the developer's experience level
- **PR review bot** — posts inline comments on GitHub PRs with suggested code fixes; triggered by webhooks or `/review` comments
- **Automation** — push to main → commit status check; PR merged → full repo health scan; score drops ≥10 points → opens a GitHub issue
- **Progress tracking** — charts how scores evolve across reviews over time
- **Scheduled scans** — daily/weekly/monthly automated repo health checks
- **RAG knowledge base** — ChromaDB + LlamaIndex backed by OWASP Top 10, clean architecture, DDIA, and testing principles
- **Observability** — all LLM calls traced in Langfuse

## Quick Start

### 1. Clone the repo

```bash
git clone https://github.com/YOUR_USERNAME/engineering-review-platform.git
cd engineering-review-platform
```

### 2. Run the setup script

**macOS / Linux:**
```bash
chmod +x setup.sh
./setup.sh
```

**Windows (PowerShell as Administrator):**
```powershell
.\setup.ps1
```

The script will:
- Check Docker and Ollama are installed
- Copy `.env.example` → `.env` and generate a random secret key
- Prompt you to fill in your GitHub token
- Pull the three required Ollama models (~8 GB total)
- Start all 10 Docker services

### 3. Open the app

| Service | URL |
|---------|-----|
| Frontend UI | http://localhost:3000 |
| API (Swagger) | http://localhost:8000/docs |
| Langfuse (observability) | http://localhost:3001 |

Default login: **admin@local.dev** / **Admin1234!** (change in `.env`)

```bash
# 1. Copy and edit env file
cp .env.example .env
# Fill in GITHUB_TOKEN, GITHUB_WEBHOOK_SECRET, etc.

# 2. Pull Ollama models
ollama pull qwen2.5-coder:7b
ollama pull mistral:7b
ollama pull nomic-embed-text

# 3. Start everything
docker compose up -d
```

## GitHub Integration (PR Review Bot)

To enable automatic PR reviews and commit status checks:

1. [Create a GitHub PAT](https://github.com/settings/tokens) with scopes: `repo`, `pull_requests:write`
2. Add it to `.env` as `GITHUB_TOKEN`
3. Install [ngrok](https://ngrok.com/) and run: `ngrok http 8000`
4. In your GitHub repo → Settings → Webhooks → Add webhook:
   - **Payload URL:** `https://xxxx.ngrok-free.app/webhooks/github`
   - **Content type:** `application/json`
   - **Secret:** your `GITHUB_WEBHOOK_SECRET` value from `.env`
   - **Events:** Pull requests, Pushes, Issue comments

Full webhook and automation guide: [`STARTUP.md`](STARTUP.md)

## Configuration

All config lives in `.env`. Key variables:

| Variable | Description |
|----------|-------------|
| `GITHUB_TOKEN` | PAT for posting PR comments and commit statuses |
| `GITHUB_WEBHOOK_SECRET` | Shared secret between GitHub and this app |
| `ADMIN_EMAIL` | UI login email |
| `ADMIN_PASSWORD` | UI login password |
| `SECRET_KEY` | JWT signing key (auto-generated by setup script) |
| `OLLAMA_PRIMARY_MODEL` | Main code review LLM (default: `qwen2.5-coder:7b`) |
| `OLLAMA_SYNTHESIS_MODEL` | Report synthesis LLM (default: `mistral:7b`) |

## Stopping / Resetting

```bash
# Stop all services (data preserved)
docker compose down

# Stop and delete all data
docker compose down -v

# Rebuild after code changes
docker compose build api celery_worker
docker compose up -d
```

## Guardrails

This system is designed with explicit human-in-the-loop safeguards:

- Never auto-merges PRs
- Never posts `APPROVE` reviews — only `COMMENT`
- All automated comments include a notice that a human must make the final decision
- Score regression issues are flagged, not acted upon
